# AI 에이전트 기반 카탈로그 품질 관리 워크플로

이 문서는 카탈로그 검사 도구에서 발견되는 오류를 GitHub 이슈로 자동 등록하고, AI 에이전트(Copilot 코딩 에이전트 · Claude Code 등)의 도움을 받아 이를 해결하는 절차를 설명합니다. 아울러 통폐합이 잦은 저축은행 업계를 대상으로, AI 에이전트에게 업계 뉴스 조사를 위임하는 워크플로도 함께 다룹니다.

전체 철학은 [사이트 헬스 워크플로](SITE_HEALTH_WORKFLOW.md)와 동일합니다 — **검출(detect)은 자동, 최종 판정과 카탈로그 반영(merge)은 사람**이 합니다. AI 에이전트는 검출·조사·초안 작성을 돕는 도구이지 판결자가 아닙니다.

## 1. 카탈로그 품질 자동 검사 (`catalog-quality.yml`)

카탈로그 검사 도구를 주기적으로 실행하여 발견된 오류를 하나의 추적(tracking) 이슈로 자동 등록/갱신합니다.

### 트리거

- **스케줄**: 매주 월요일 00:00 UTC (한국시간 월요일 09:00)
- **수동 실행**: Actions 탭에서 `workflow_dispatch`
  - `assign_copilot` 옵션을 켜면 생성/갱신된 추적 이슈를 Copilot 코딩 에이전트에 배정합니다.
- **푸시**: `docs/Catalog.xml` 또는 `docs/images/**` 변경이 `main`에 반영될 때

### 동작

1. [`checkimages.cs`](../src/checkimages.cs)로 누락/미사용 로고 이미지를 검출합니다. (오프라인, 결정론적)
2. [`catalogutil.cs`](../src/catalogutil.cs)로 XML 스키마 검증 및 URL 접속성 점검을 수행합니다.
3. 두 도구 중 하나라도 문제를 보고하면(비정상 종료 코드), 검사 리포트를 담은 **추적 이슈**를 생성하거나 기존 이슈를 갱신합니다.
4. 문제가 모두 해소되면 열려 있던 추적 이슈를 자동으로 닫습니다.

이슈는 `catalog-quality` 라벨로 관리되며, **열려 있는 추적 이슈는 항상 최대 1건**입니다(중복 등록 방지). 이슈 본문에는 도구 출력 전문과 권장 조치, 재검증 명령이 포함되어 AI 에이전트가 바로 후속 작업에 착수할 수 있습니다.

### AI 에이전트로 해결하기

추적 이슈를 Copilot 코딩 에이전트에 배정하거나, 이슈 본문을 Claude Code에 전달하세요. 이슈 본문의 권장 조치를 따라:

- **누락된 이미지**: 해당 기관 공식 로고를 `docs/images/<Category>/<Id>.png`로 추가 ([`fetchfavicon.cs`](../src/fetchfavicon.cs) 활용).
- **미사용 이미지**: 삭제된 서비스의 잔여 이미지라면 함께 제거.
- **스키마 오류**: `Catalog.xml`을 `Catalog.xsd`에 맞게 수정.

수정 후 재검증:

```bash
dotnet run --file src/checkimages.cs -- ./docs/Catalog.xml ./docs/images
dotnet run --file src/catalogutil.cs -- ./docs/ ./outputs/
```

## 2. 저축은행 통폐합 점검 (`savings-bank-merger-watch.yml`)

저축은행 업계는 인수합병·통폐합·명칭변경·영업정지가 잦습니다. 이 워크플로는 매월 AI 에이전트에게 **"최근 저축은행 통폐합 뉴스"를 조사**하도록 위임하는 이슈를 자동 생성합니다.

### 트리거

- **스케줄**: 매월 1일 00:00 UTC (한국시간 매월 1일 09:00)
- **수동 실행**: Actions 탭에서 `workflow_dispatch` (`assign_copilot` 옵션 지원)

### 동작

1. `Catalog.xml`에서 이름에 "저축은행"이 포함된 서비스 항목을 모두 추출합니다.
2. 추출한 목록과 AI 에이전트 작업 지침을 담은 **조사 요청 이슈**를 생성합니다. (`savings-bank-merger-watch` 라벨)
3. 같은 달에 이미 조사 이슈가 열려 있으면 중복 생성하지 않습니다.

### AI 에이전트의 역할

이슈에 배정된 AI 에이전트는 다음을 수행합니다.

1. 카탈로그의 저축은행 목록을 기준으로 최근(지난 3~6개월) 국내 저축은행 통폐합/인수합병/명칭변경/폐업 뉴스를 검색 (금융감독원, [저축은행중앙회](https://www.fsb.or.kr/), 경제 뉴스 등).
2. 확인된 변경 건마다 대상 기관, 변경 유형, **출처 URL**, 시점, 반영 방향(URL 교체 / 명칭·Id 갱신 / 삭제 / 신규 추가)을 정리.
3. [`checksites.cs`](../src/checksites.cs) `probe`로 대상 사이트 상태를 검증.
4. 근거(출처 URL 포함)와 함께 이슈에 제안하고, 승인 후 PR로 처리.

> **자동 삭제 금지**: 폐업·통폐합으로 항목 삭제가 필요한 경우 반드시 사람의 승인을 받으세요.

## 자동 vs 사람 판정 경계선

| 단계 | 자동 | 사람/승인 필요 |
|------|:---:|:---:|
| 도구 실행 및 문제 검출 | ✓ | |
| 이슈 등록/갱신/종료 | ✓ | |
| 뉴스 조사·초안 작성 (AI) | ✓ | |
| 이미지 추가 (누락 보완) | ✓ | |
| 항목 삭제 / Id 변경 / 통폐합 반영 | | ✓ |
| PR 병합(merge) | | ✓ |
